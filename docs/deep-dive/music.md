# Deep dive: music generation

Every song on the network is generated by a local text-to-music model from a
positive-only style caption, quality-gated before it can air, and filed into a
per-show genre lane. The scheduler then keeps it from ever sounding like a shuffle
(covered in [the station engine](station-engine.md); summarized here from the
music side).

```
caption (positive-only) ─► text-to-music model (locked recipe) ─► FLAC ─► quality gate ─► lane pool
   + lyrics or [instrumental]     (single-flight, one at a time)         (loudness/peak/    (candidate
   + genre tag                                                            duration/valid)     -> approved)
```

## 1. The model and the locked recipe

The music engine is a **self-hosted text-to-music foundation model** (this build
uses [ACE-Step](https://github.com/ace-step/ACE-Step)) running as an HTTP service
on its own GPU node. A thin client posts a caption and gets audio back. A health
check gates every run so an unreachable server aborts rather than producing
silence.

The recipe is **locked** (the same values used everywhere), because the defaults
drift:

| Parameter | Value | Why |
|---|---|---|
| `guidance_scale` (CFG) | `8.0` | Stable and caption-adherent. |
| `inference_steps` | `40` | Fidelity for an offline render. |
| `thinking` | `false` | The model's helper LM rewrites your caption and drifts the genre. A "jazz-hop boom-bap" prompt drifted to "generic pure jazz" with it on. |
| `audio_format` | `flac` | Lossless master. |
| `duration` | ~`200 s` | (Range roughly 10 to 600.) |
| `seed` | `-1` | Random. |

Footgun worth knowing: the raw client defaults are weaker (`guidance` coerced to
`7.0`, `25` steps, `thinking` on). Always set the recipe explicitly. An older
generator that used the defaults plus a random guidance range was the original
cause of the genre-drift bug.

## 2. Positive-only captions

The conditioning text is **style, mood, and instrumentation stated in the
affirmative**: genre, a vibe sentence, a vocal descriptor, a couple of textures,
and a quality tag like "tight, well produced, distinct and memorable." Captions are
capped (around `512` chars). You describe what you want, never what you do not,
with one deliberate exception below.

## 3. The "never name the genre you are avoiding" rule

Some words prime the model to drift toward an unwanted genre even inside a
negation. So a **quarantine list is enforced before generation**: the caption
(joined with its lyrics) is scanned against a trigger set, for example

```
country, nashville, americana, twang, southern drawl, pedal steel, banjo, cowboy
```

and any hit **blocks the track** (raises, never generates). To steer away from a
genre you phrase the constraint positively instead (e.g. "no country influence" is
fine as a texture line; naming the genre to forbid it is not). The lesson: with a
diffusion-style music model, mentioning a genre at all pulls toward it.

## 4. Single-flight generation

Generation is **serialized behind one exclusive lock** (an OS advisory file lock).
The inference backend is single-flight: concurrent inference on the shared node is
unstable, so only one track generates at a time across the whole network.
Consequences baked in:

- The health-check timeout is deliberately generous (about `8 s`), because a node
  busy generating for another station must not be misread as "down". A false "down"
  once made a station exit and sleep for over an hour.
- **You scale throughput by adding nodes, not concurrency.**

## 5. Vocal vs instrumental (the 422 rule)

Every track is tagged, and the `lyrics` field is mandatory in a specific way:

- **Vocal** -> `instrumental=false` and `lyrics` must be a non-empty string. A null
  or missing lyrics field on a vocal request returns **HTTP 422**.
- **Instrumental** -> `instrumental=true` and `lyrics` set to the sentinel
  `"[instrumental]"` (never null).

Every track is also given a coarse `genre` tag, because the scheduler's
back-to-back genre-spacing rule depends on it.

## 6. Per-show lanes

Each show or daypart owns a **genre lane** with its own tuned caption pool. A lane
carries a set of sub-genres (each genre-tagged so spacing works), a vibe sentence,
a pool of vocal descriptors, a texture pool (a couple sampled per track), and word
pools for the fictional artist and title names. A morning show's lane might be
bright soul-pop and breakfast disco-pop; a drive-time show's lane fast dance-punk
and electroclash; an overnight lane nocturnal synth-pop and cold wave. Continuous
sister stations use the same recipe with their own sub-lanes (a lo-fi channel split
into morning/study/evening/night, a yacht channel into day yacht-rock and night
yacht-house).

## 7. The quality gate

A separate automated gate runs `ffprobe` plus one `ffmpeg` pass
(`ebur128=peak=true,astats`) over each FLAC. Strict pass or fail:

- **Valid stream**: codec `flac`, `sample_rate > 0`, `channels > 0` (catches corrupt
  output).
- **Duration floor**: about `45 s` minimum (catches silence and collapsed
  generations).
- **Peak ceiling**: sample peak no hotter than about `-1.0 dBFS` (catches clipping).
- **Loudness band**: integrated LUFS within a per-lane target, plus or minus about
  `3 LUFS` (e.g. a morning lane around `-16`, an evening lane around `-18`, a
  late-night lane lower).

Failures are **moved to a dated quarantine folder** with their sidecar, never
silently kept. Passes get a `.qa.json` sidecar recording measured peak, RMS,
duration, and loudness. The gate exits non-zero if anything failed.

## 8. How tracks are chosen for air

The scheduler keeps generated music from sounding like a shuffle by enforcing
three rules at once (full algorithm in [the station engine](station-engine.md)):

1. **No song repeats too soon**: a rotation gap (about `16 h` by default, tighter
   for small special-show libraries).
2. **No two of the same style back to back**: genre spacing (the last genre and the
   last couple of artists are avoided), using the `genre` tag from generation.
3. **No lane bleed**: a show exhausts its own lane and a short whitelist of
   compatible lanes before reaching anywhere else; genre-locked specials borrow
   from no one.

## 9. Backfill, do not wipe

Regeneration is slow and single-flight, and lanes are sized near one-day rotation,
so **emptying a lane all at once makes the show play silence** before replacements
exist. The operational rule is: backfill the currently-airing show first, and never
wipe a lane wholesale. This is enforced structurally:

- Generators are **target-and-cap top-ups** (`while count < target`, capped new
  tracks per run as a review checkpoint), not replacements.
- New tracks land as `candidate` and do not air until promoted to `approved`, so a
  regeneration run never reduces the live inventory.
- The QA gate quarantines rejects rather than deleting in place.

Music generation is deliberately manual and non-autonomous: the scheduler only
selects, libraries grow only when a generator is run on purpose.

## 10. Output

Each track is a **FLAC plus a JSON sidecar** of the same stem, carrying
`review_status`, `genre`, `show_lane` / `show_id`, `artist`, `title`, `caption`,
`instrumental`, the exact `recipe` used, and generation timestamps. Passing tracks
additionally get a `.qa.json` with the measured loudness and peak.
